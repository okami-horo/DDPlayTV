# player-danmaku-line-limit-tv-selector Specification

## Purpose

在 TV 播放界面的“弹幕参数设置”中，用户可以通过遥控器切换来配置“弹幕行数限制”，无需数字输入或额外确认；配置切换后立即生效并持久化（范围为 `0..12`，其中 `0` 为无限制）。

## ADDED Requirements

### Requirement: TV 端行数限制可切换且立即生效

系统 SHALL 在 TV UI mode 下提供可用 DPAD 左右切换的“弹幕行数限制”控件；用户每次切换后，系统 SHALL 立即刷新弹幕渲染的最大行数限制并持久化该配置。

#### Scenario: TV 上切换滚动弹幕行数限制立即生效

- **WHEN** 用户在播放界面打开“弹幕参数设置”
- **AND** 用户将焦点移动到“滚动弹幕行数限制”控件
- **AND** 用户通过 DPAD 左右切换行数限制（范围 `0..12`，`0`=无限制）
- **THEN** 弹幕渲染的最大行数限制应立即更新
- **AND** 用户关闭设置并再次进入时应展示该次切换后的值

### Requirement: 三种弹幕类型的行数限制分别配置

系统 SHALL 对滚动/顶部/底部弹幕分别维护独立的行数限制配置；用户在不同类型间切换时，系统 SHALL 展示并编辑对应类型的行数限制值，且互不影响。

#### Scenario: 顶部弹幕行数限制不影响滚动弹幕

- **WHEN** 用户在“弹幕参数设置”中将“顶部弹幕行数限制”切换为某个值
- **AND** 用户切换回“滚动弹幕”模式
- **THEN** “滚动弹幕行数限制”应保持其原有配置值不变

### Requirement: TV 端不依赖数字输入框完成配置

系统 SHALL 在 TV UI mode 下避免将“弹幕行数限制”暴露为可编辑的数字输入框交互；用户无需唤起软键盘即可完成配置与生效。

#### Scenario: TV 端焦点导航不进入数字输入

- **WHEN** 用户使用 DPAD 在“弹幕参数设置”中导航到“弹幕行数限制”
- **THEN** 用户可通过左右切换完成配置
- **AND** 不需要聚焦到数字输入框或唤起软键盘即可完成生效与持久化
